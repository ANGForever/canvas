模型上下文协议 (MCP) 解析

MCP 是一种开放协议，旨在标准化应用程序向大型语言模型 (LLM) 提供上下文和工具的方式，是构建统一智能设备控制中枢的关键。

用户请求到设备响应的交互流程

1. 用户发出指令 (例如 "卧室灯调暗到30%")
2. 请求处理引擎 (LLM) 解析意图与参数
3. MCP 工具路由/选择器选择合适工具
4. MCP 客户端交互层发送 `tools/call`
5. MCP 服务器执行设备操作
6. MCP 服务器返回结果
7. 响应聚合引擎格式化并输出响应

此流程清晰地展示了 SynHome Core 如何处理用户请求并通过 MCP 与智能设备交互，最终完成用户指令并提供反馈。

MCP 工具模式示例 (inputSchema)

清晰的 MCP 工具 `inputSchema` (使用 JSON Schema 定义) 对于 LLM 准确理解和调用至关重要。以下为简化示例：

灯光亮度设置

{
  "type": "object",
  "properties": {
    "device_id": {"type": "string"},
    "brightness": {"type": "integer", "min":0, "max":100}
  },
  "required": ["device_id", "brightness"]
}

恒温器温度设置

{
  "type": "object",
  "properties": {
    "device_id": {"type": "string"},
    "temperature": {"type": "number"},
    "unit": {"type": "string", "enum": ["celsius", "fahrenheit"]}
  },
  "required": ["device_id", "temperature"]
}

这些模式中的 `description` 字段（未完全展示）对 LLM 理解工具功能和适用场景至关重要。通过标准化模式，SynHome Core 可以更智能地将用户自然语言请求映射到具体的设备操作。
